# RP2040 (Arduino mbed) — build, upload, serial

Board package: **arduino:mbed_rp2040**

Pico FQBN: **`arduino:mbed_rp2040:pico`**

Nano RP2040 Connect FQBN: **`arduino:mbed_rp2040:nanorp2040connect`**

Install the core once:

```bash
arduino-cli core update-index
arduino-cli core install arduino:mbed_rp2040
```

Install USB permissions once (required for upload from BOOTSEL):

```bash
sudo ~/.arduino15/packages/arduino/hardware/mbed_rp2040/4.6.0/post_install.sh
```

That adds udev rules for Raspberry Pi / Arduino USB devices (`2e8a`, etc.).

---

## Which port is the Pico?

| What you see | What it is |
|---|---|
| `/dev/ttyS0`, `/dev/ttyS1`, … | On-board PC serial ports. **Not the Pico.** |
| `/dev/ttyACM0` | Pico running a sketch (USB serial). **Use this for monitor and normal upload.** |
| USB `2e8a:00c0` | Pico running (Arduino identifies it correctly) |
| USB `2e8a:0003` + drive `RPI-RP2` | Pico in **BOOTSEL** bootloader mode (no serial port) |

Check what is connected:

```bash
arduino-cli board list
lsusb | grep -i 2e8a
```

## Normal workflow (sketch already on the board)

Compile:

```bash
arduino-cli compile --fqbn arduino:mbed_rp2040:pico SKETCH_FOLDER
```

Upload (Pico must show as `/dev/ttyACM0`):

```bash
arduino-cli upload -p /dev/ttyACM0 --fqbn arduino:mbed_rp2040:pico SKETCH_FOLDER
```

The mbed core sends a 1200-baud reset over USB serial, then uploads with `rp2040load`.

Serial monitor:

```bash
arduino-cli monitor -p /dev/ttyACM0 -c baudrate=9600
```

## BOOTSEL upload (hold BOOTSEL while plugging in)

The Pico appears as a 128 MB USB drive named **RPI-RP2**, not a serial port.

After udev rules are installed:

```bash
arduino-cli upload --fqbn arduino:mbed_rp2040:pico SKETCH_FOLDER
```

No `-p` flag — upload talks to the bootloader directly.

---

## Manual UF2 fallback (always works)

Use this if `arduino-cli upload` fails or you have not run `post_install.sh` yet.

1. Put the Pico in BOOTSEL (hold **BOOTSEL**, plug in USB, release).
2. Compile the sketch (see above).
3. Convert the built ELF to UF2:

```bash
SKETCH=mpu6050_rp2040
FQBN=arduino:mbed_rp2040:pico

arduino-cli compile --fqbn "$FQBN" "$SKETCH"

ELF=$(find ~/.cache/arduino/sketches -name '*.ino.elf' -printf '%T@ %p\n' | sort -n | tail -1 | cut -d' ' -f2-)
~/.arduino15/packages/arduino/tools/rp2040tools/1.0.6/elf2uf2 "$ELF" /tmp/firmware.uf2
```

4. Mount the drive if it is not mounted automatically:

```bash
udisksctl mount -b /dev/disk/by-id/usb-RPI_RP2*
```

5. Copy the UF2 file. The Pico reboots when the copy finishes:

```bash
cp /tmp/firmware.uf2 /run/media/$USER/RPI-RP2/
sync
```

6. After reboot, the Pico should appear on `/dev/ttyACM0`.

## Troubleshooting

**`arduino-cli board list` only shows `/dev/ttyS0`**

The Pico is not connected, not running USB serial, or is in BOOTSEL. Unplug/replug; check `lsusb`.

**Upload fails with picotool / permission errors**

Run `post_install.sh` (see above), unplug/replug, or use the UF2 fallback.

**`No accessible RP2040 devices in BOOTSEL mode were found` but `lsusb` shows `2e8a:0003`**

Permissions issue. Run `post_install.sh` or use the UF2 fallback (mount + copy).

**Custom I2C pins on mbed (e.g. GP16/GP17)**

Do not use `Wire.setSDA()` / `Wire.setSCL()` — they do not exist on mbed. Do not name variables `I2C_SDA` / `I2C_SCL` (macros conflict). Create a separate bus:

```cpp
arduino::MbedI2C myWire(16, 17);  // SDA, SCL
myWire.begin();
```

Default `Wire` on Pico uses GP4 (SDA) and GP5 (SCL).
